# 快速开始指南

## 版本说明

| 版本 | 说明 | 适用场景 |
|------|------|----------|
| 标准版 | 需要运行 install.bat 安装 | 网络正常的用户 |
| 完整版 | 含虚拟环境和模型，开箱即用 | 网络不便或经常重装系统 |

---

## 标准版 - Windows 用户

### 一键安装（使用国内镜像，无需 VPN）
```
双击 scripts\install.bat
```

### 一键启动
```
双击 scripts\start.bat
```

---

## 完整版打包（含虚拟环境和模型）

如果你想分发一个**开箱即用**的版本给客户：

```
1. 先运行 scripts\install.bat 安装所有依赖
2. 准备好 6 个音色 WAV 文件
3. 运行 scripts\pack_full.bat 生成完整打包
4. 将生成的 ZIP 发给客户
```

客户使用方式：
```
1. 解压 ZIP
2. 双击"首次使用-安装前端依赖.bat"（仅首次）
3. 双击"启动.bat"
```

---

## macOS / Linux 用户

### 一键安装
```bash
chmod +x scripts/*.sh
./scripts/install.sh
```

### 一键启动
```bash
./scripts/start.sh
```

---

## 虚拟环境说明

安装脚本会自动在 `backend/venv` 目录创建 Python 虚拟环境。

**重装系统后恢复步骤：**
1. 安装 Python 3.10+ 和 Node.js 18+
2. 重新运行安装脚本
3. 音色文件和模型文件如果之前备份了可直接复制回来

**手动激活虚拟环境：**
```bash
# Windows
cd backend
venv\Scripts\activate

# macOS/Linux
cd backend
source venv/bin/activate
```

## 常见问题

**Q: 重装系统后需要重新下载模型吗？**
A: 如果备份了 `backend/checkpoints` 目录，直接复制回来即可，无需重新下载。

**Q: 虚拟环境损坏怎么办？**
A: 删除 `backend/venv` 目录，重新运行安装脚本。

**Q: 如何更新项目？**
A: 
```bash
git pull
# 然后重新运行安装脚本
```
